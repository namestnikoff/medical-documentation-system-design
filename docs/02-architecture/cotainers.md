# Компонентная структура

- Frontend: интерфейс врача, регистратора, личный кабинет пациента
- API Gateway: маршрутизация и единая точка доступа
- Сервис пациентов: ведёт карточки пациентов, поиск, историю
- Сервис расписаний: график работы, слоты, запись
- Сервис приёмов: логика медицинского обслуживания — прием, назначение, закрытие
- Сервис медицинских документов: формирование, хранение, подписание, отправка в ЕГИСЗ
- Сервис пользователей: учёт медработников, управление ролями, интеграция с ЕСИА
- Сервис платных услуг: прейскурант, оплата, ДМС
- Сервис статистики: отчёты, выгрузки по формам, анализ
- Сервис аудита: лог действий пользователей
- Интеграция: ЕГИСЗ, ЕСИА, платежи, федеральные справочники
- Хранилище: PostgreSQL (для всех сервисов)
