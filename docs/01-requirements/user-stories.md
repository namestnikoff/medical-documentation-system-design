# Пользовательские истории (User Stories)

## Обзор

Пользовательские истории описывают функциональность системы с точки зрения конечных пользователей. Формат: **Как [роль], я хочу [действие], чтобы [цель/выгода]**.

## Врач

**US-001: Быстрый доступ к истории болезни**
- **Как** врач
- **Я хочу** найти пациента по ФИО или СНИЛС за 1-2 секунды
- **Чтобы** не тратить время приёма на поиск нужной медицинской карты

**US-002: Оформление приёма**
- **Как** врач
- **Я хочу** внести данные осмотра, поставить диагноз и назначить лечение в одной форме
- **Чтобы** быстро оформить приём и не переключаться между разными экранами

**US-003: Электронная подпись**
- **Как** врач
- **Я хочу** подписать медицинский документ электронной подписью за 5 секунд
- **Чтобы** документ автоматически отправился в ЕГИСЗ без дополнительных действий

**US-004: Шаблоны для типовых случаев**
- **Как** врач
- **Я хочу** использовать готовые шаблоны для типовых диагнозов (ОРВИ, гипертония)
- **Чтобы** сократить время на заполнение документа с 10 минут до 2 минут

## Регистратор

**US-005: Запись пациента на приём**
- **Как** регистратор
- **Я хочу** видеть свободные слоты времени у всех врачей нужной специальности
- **Чтобы** записать пациента на удобное время за 1-2 минуты

**US-006: Регистрация нового пациента**
- **Как** регистратор
- **Я хочу** быстро создать медицинскую карту, проверив отсутствие дубликатов
- **Чтобы** избежать повторной регистрации одного пациента

**US-007: Оформление платной услуги**
- **Как** регистратор
- **Я хочу** выбрать услугу из прейскуранта, принять оплату и выдать чек
- **Чтобы** быстро оформить платный приём без задержки пациента

## Кабинет медицинской статистики (КМС)

**US-008: Автоматическая статистика**
- **Как** сотрудник КМС
- **Я хочу** формировать месячный отчёт по форме №30 одним нажатием кнопки
- **Чтобы** не тратить 3-4 дня на ручной сбор данных из карт

**US-009: Контроль качества документов**
- **Как** сотрудник КМС
- **Я хочу** видеть список некорректно заполненных документов
- **Чтобы** уведомить врачей о необходимости исправления

## Руководство

**US-010: Мониторинг работы поликлиники**
- **Как** главный врач
- **Я хочу** видеть дашборд с ключевыми показателями (количество приёмов, загрузка врачей, финансы)
- **Чтобы** оперативно принимать управленческие решения

## Пациент

**US-011: Онлайн-запись на приём**
- **Как** пациент
- **Я хочу** самостоятельно записаться на приём к врачу через сайт или мобильное приложение
- **Чтобы** не тратить время на звонок в регистратуру или очередь
- **Обоснование**: Снижает нагрузку на регистратуру, удобно для 70-80% пациентов

**US-012: Доступ к своим медицинским документам**
- **Как** пациент
- **Я хочу** просмотреть свои анализы, выписки и результаты обследований через личный кабинет
- **Чтобы** не приезжать в поликлинику за бумажными копиями
- **Примечание**: Частично покрывается через Госуслуги после отправки СЭМД в ЕГИСЗ

## Приоритизация

### Критический приоритет (MVP)
Система не может функционировать без этих историй:
- US-001: Поиск пациента
- US-002: Оформление приёма
- US-003: Электронная подпись
- US-005: Запись на приём (регистратором)
- US-006: Регистрация пациента

### Высокий приоритет (Release 1)
Необходимо для полноценной работы:
- US-007: Платные услуги
- US-008: Автоматическая статистика
- US-011: Онлайн-запись пациентами (снижает нагрузку на регистратуру на 50-70%)

### Средний приоритет (Release 2)
Улучшает эффективность работы:
- US-004: Шаблоны документов
- US-009: Контроль качества
- US-010: Дашборд для руководства
- US-012: Личный кабинет с документами (дополнительно к Госуслугам)

## Обоснование приоритета онлайн-записи

**Почему US-011 в высоком приоритете:**
1. **Тренд**: По данным 2024-2025, 60-80% записей в крупных поликлиниках через интернет
2. **Разгрузка регистратуры**: Снижение нагрузки на 50-70%, регистраторы освобождаются для других задач
3. **Удобство**: Запись 24/7 без ожидания в очереди или звонка
4. **Ожидания пациентов**: Современный стандарт обслуживания
5. **Интеграция с Госуслугами**: Возможна запись через ЕСИА

**Почему US-012 в среднем приоритете:**
- Базовый функционал уже есть через Госуслуги (СЭМД автоматически доступны)
- Дополнительный личный кабинет — улучшение UX, но не критично для MVP
