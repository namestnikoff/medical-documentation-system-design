# Трассировка требований

## Обзор

Трассировка требований показывает связь между бизнес-требованиями, функциональными и нефункциональными требованиями системы.

## Основные связи

### Бизнес-требования → Функциональные требования

**BR-001: Автоматизация документооборота**
- FR-001: Регистрация и поиск пациентов
- FR-003: Ведение приемов
- FR-005: Создание СЭМД
- FR-006: Электронная подпись

**BR-002: Соответствие законодательству**
- FR-005: Создание СЭМД (формат CDA R2)
- FR-006: Электронная подпись (УКЭП)
- FR-008: Интеграция с ЕГИСЗ

**BR-003: Учет платных услуг**
- FR-009: Оформление и оплата платных услуг
- FR-010: Учет договоров ДМС

**BR-004: Медицинская статистика**
- FR-011: Сбор и формирование отчетности
- FR-012: Контроль качества документации

**BR-005: Безопасность персональных данных**
- FR-013: Управление пользователями и правами
- FR-016: Журнал аудита

### Функциональные требования → Нефункциональные требования

**Производительность**
- FR-001, FR-007 → NFR-001 (время поиска)
- FR-003, FR-005 → NFR-001 (создание документа)
- Все FR → NFR-002 (пропускная способность)

**Надёжность**
- FR-008 → NFR-004 (отказоустойчивость интеграций)
- Все FR → NFR-003 (доступность системы)

**Масштабируемость**
- FR-003, FR-005 → NFR-005, NFR-006 (рост нагрузки)

**Безопасность**
- FR-013 → NFR-007 (аутентификация и авторизация)
- FR-016 → NFR-009 (аудит действий)
- FR-006 → NFR-010 (электронная подпись)
- Все FR с ПДн → NFR-008 (шифрование)

**Соответствие стандартам**
- FR-005, FR-008 → NFR-013 (требования ЕГИСЗ)
- Все FR → NFR-013 (152-ФЗ, 323-ФЗ)
